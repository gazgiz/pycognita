## Reporting a Vulnerability

We take the security of our software seriously. If you believe you have found a security vulnerability in PyCognita, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to:

**inquiry@cleverplant.com**

You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

We prefer all communications to be in English or Korean.

### What to Include

Please include as much of the following information as possible in your report:

*   The type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
*   Full paths to source file(s) related to the manifestation of the issue
*   The location of the affected source code (tag/branch/commit or direct URL)
*   Any special configuration required to reproduce the issue
*   Step-by-step instructions to reproduce the issue
*   Proof-of-concept or exploit code (if possible)
*   Impact of the issue, including how an attacker might exploit the issue

### Disclosure Policy

*   We ask that you do not disclose the issue to the public or any third party until we have had a reasonable opportunity to resolve it.
*   We will investigate the issue and verify its validity.
*   We will notify you when we have fixed the issue.
